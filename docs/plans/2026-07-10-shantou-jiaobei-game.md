## 1. 背景与文化理解

### 1.1 什么是"圣杯"（筊杯 / Jiaobei）

筊杯（潮汕称"跋杯"）是潮汕地区民间信仰中与神明沟通的占卜器具，由 **两片半月形** 木/竹片组成。每片有 **一凸面（阳/正面，圆弧隆起）** 和 **一平面（阴/反面，平整）**。信众向神明禀报所求后，将两片筊杯掷于地面，依据落地后的正反组合获得神明指示。

### 1.2 三种杯象（物理朝向 + 含义）

| 杯象 | 两片朝向 | 含义 | 游戏中表达 |
|------|----------|------|-----------|
| 🏆 **圣杯（阳杯）** | 一凸一平（一正一反） | 神明**同意 / 肯定 / 可行** | ✅ 吉 — 所愿可成 |
| 😄 **笑杯** | 两片皆平面朝上 | 神明**微笑**：问题不清 / 未定 / 再想想 | 🤔 待定 — 需重新表述或再问 |
| 🌑 **阴杯（哭杯）** | 两片皆凸面朝上 | 神明**不同意 / 否定** | ❌ 否 — 时机未到或不可行 |

### 1.3 乞愿（跋杯）流程

1. **上香禀报**：双手合十，报上姓名 / 生辰 / 住址
2. **陈述所求**：清楚说明所问之事（心愿 / 疑问）
3. **掷杯问询**：双手捧筊杯至胸前 → 轻掷于地
4. **连掷三杯**：通常需 **连续三个圣杯** 方为神明首肯
5. **谢恩**：得圣杯后答谢

> **游戏映射**：玩家输入心愿 → 摄像头识别"请���姿势"（双手合十）→ 释放掷杯 → Babylon.js 物理引擎模拟落地 → 判定杯象 → 给出解答 → 三掷判定。

## 2. 目标与非目标

### 2.1 目标
- 静态站点，可部署到 **Gitee Pages**（纯前端、无后端、相对路径）。
- 站点架构支持 **后续多游戏扩展**（多 game 模块化）。
- 首个游戏：潮汕圣杯占卜，核心体验 = **摄像头请愿姿势识别 + Babylon.js 物理掷杯 + 三象解答**。
- 文化准确、体验有仪式感、视觉有潮汕庙宇韵味。

### 2.2 非目标（本期不做）
- 不做后端 / 数据库 / 账号系统。
- 不做付费、广告、社交分享（可后续扩展）。
- 摄像头仅做**姿势触发**（双手合十），不做完整人体姿态连续追踪（用 MediaPipe Hands/Pose 轻量检测即可）。
- 不承诺 100% 文化准确性认证，标注"娱乐用途"。

## 3. 技术选型

| 维度 | 选型 | 理由 |
|------|------|------|
| 框架 | **Next.js**（App Router，`output: 'export'` 静态导出） | 用户指定；后续可转型动态站；生态成熟 |
| 语言 | **TypeScript** + **React** | Next.js 原生；可维护、多游戏扩展友好 |
| 渲染/物理 | **Babylon.js**（+ Havok 或 Ammo.js 物理插件） | 用户指定；物理引擎强，3D 效果好 |
| 姿势识别 | **MediaPipe Hands**（@mediapipe/tasks-vision，TS/ESM） | 浏览器原生、轻量、识别双手合十 |
| 语音输入 | **Web Speech API** `SpeechRecognition`（中文 `zh-CN`） | 浏览器原生、免依赖；不支持时文本兜底 |
| 音效 | 免费 SFX（freesound.org 等）→ Web Audio API | 落地声、仪式背景音 |
| 部署 | Gitee Pages（发布 `out/` 静态目录） | 纯静态、HTTPS 由平台提供 |
| 摄像头 | `getUserMedia` + `<video>` + MediaPipe | 标准浏览器 API |

> **静态导出关键**：`next.config.mjs` 设 `output: 'export'`、`images.unoptimized: true`；游戏页用客户端组件（`'use client'`）承载 Babylon/MediaPipe/Web Speech（均为浏览器 API，禁 SSR）。`base` 子路径暂不配，部署时再加 `assetPrefix`/`basePath`。

### 5.2 游戏流程（用户旅程）

```
[首页卡片] → [IntroScene 文化介绍]
          → [WishScene 输入心愿（文本 或 语音识别）+ "所愿"展示]
          → [OfferingScene 请愿掷杯 ×3]
                1. 请求摄像头权限
                2. <video> 预览 + MediaPipe 检测双手
                3. 引导玩家"双手合十"（请愿姿势）
                4. 识别到姿势 N 秒 → "心诚则灵" → 释放掷杯
                5. Babylon.js 物理模拟两片筊杯下落、碰撞、翻转、静止
                6. ResultDetector 判定朝向 → 圣杯/笑杯/阴杯
                7. 记录本次结果，若未满三掷则循环
          → [ResultScene 三掷汇总 + 解答]
                - 三圣杯 = 神明首肯 ✅
                - 含笑杯 = 问题未明 🤔（建议重述）
                - 含阴杯 = 神明未允 ❌（建议暂缓）
                - 给出对应文案 + 仪式感谢恩语
```

### 5.3 物理与判定（核心）

**筊杯 3D 模型**：
- 半月形（肾形），用 Babylon Mesh 拼合或简单几何体近似。
- **关键**：每片有两个稳定朝向 —— 凸面朝上（阳） vs 平面朝上（阴）。通过 **质心偏移 + 底面形状** 让物理落地后自然倾向这两种稳定态。
- Havok/Ammo 物理体：两片筊杯各有刚体，给定初始位置/旋转/角速度模拟"掷出"。

**投掷动力学**（`JiaobeiThrower`）：
- 初始：两片在玩家手部位置，带向上 + 向前的初速度 + 随机角速度（模拟真实手掷）。
- 重力 + 地面碰撞 + 自身翻转。
- 静止判定：线速度 + 角速度 < 阈值持续 ~0.5s。

**结果判定**（`ResultDetector`）：
- 静止后读取每片筊杯的 **up 向量** 与世界 up 的夹角：
  - 平面朝上（平面对着天花板）→ 阴面/反
  - 凸面朝上 → 阳面/正
- 两片组合：
  - (正, 反) 或 (反, 正) → **圣杯**
  - (反, 反) 皆平面朝上 → **笑杯**
  - (正, 正) 皆凸面朝上 → **阴杯**

**随机性与公平性**：初速度/角速度含受控随机，使三象分布合理（圣杯 ~50%、笑杯 ~25%、阴杯 ~25%，接近民间经验概率）。

### 5.4 摄像头与姿势（核心）

**权限与降级**：
- 首次进入 OfferingScene 请求 `getUserMedia({ video: true })`。
- **降级方案**：若用户拒绝 / 无摄像头 / 识别失败 → 提供"点击掷杯"按钮作为替代触发（保证可玩性 & 无障碍）。

**姿势识别**（`PrayerPose`）：
- MediaPipe Hands 检测双手 21 关键点。
- 判定"双手合十"：
  1. 两只手均被检测到（handedness: Left + Right）。
  2. 双手掌心相对、指尖向上。
  3. 双手中心在画面中央偏上（胸前位置）。
  4. 两手手腕距离 / 指尖贴合度在阈值内。
- 持续保持 N 秒（进度条）→ 触发"心诚"判定 → 进入掷杯。

**隐私**：
- 摄像头流**仅在本地浏览器处理**，不上传任何画面。
- UI 明示"画面仅本地处理，不上传"。

### 5.4b 心愿语音输入（`WishSpeech`）

- 用浏览器 **Web Speech API `SpeechRecognition`**（`lang: 'zh-CN'`），麦克风识别中文心愿。
- UI：🎤 按住/点击说话 → 实时显示识别文本 → 确认"这就是我的心愿"。
- **兜底**：不支持 SpeechRecognition 的浏览器（如 Firefox / 部分移动端）→ 仅文本输入框。
- 隐私同上：语音仅在本地识别，不上传。
- 识别完成后以"所愿"卡片形式展示（毛笔字风 / Q版卷轴），强化仪式感。

### 5.5 内容与解答（`interpretations.ts`）

按 **杯象 × 心愿类别** 提供差异化文案（心愿关键词分类：感情/事业/学业/财运/健康/其他）：

- 圣杯：肯定语 + "宜趁势而行"。
- 笑杯：提示"所问未明，宜静心再思"。
- 阴杯：温和"时机未到，宜耐心"。

文案基调：**温暖、不吓人、娱乐为主**，避免封建迷信承诺。

### 5.6 视觉与氛围（Q 版卡通 + 游戏 UI 风格）

- **风格**：Q 版卡通（圆润、明亮、可爱），非写实庙宇。神明/筊杯拟人化、表情化（笑杯=憨笑表情、阴杯=嘟嘴、圣杯=开心）。
- **游戏 UI 风格**：粗描边、块状大按钮、鲜亮色块、像素/卡通字体、奖励感动效（金币飞溅、星星）。
- 色调：庙宇红 + 木色 + 金箔点缀，但提亮饱和度，整体活泼而不阴沉。
- 神坛元素：Q 版香炉、红绸、烛火（Babylon 粒子/光照）。
- **音效**（`SfxManager`，Web Audio）：
  - 掷杯落地"嗒"声（两片先后落地）。
  - 三象结果音：圣杯=清脆铃响、笑杯=俏皮"噗嗤"、阴杯=低沉"咚"。
  - 仪式背景音（可选循环、可静音）。
  - 来源：免费音效站（freesound.org 等，注意 license）。
- 掷杯瞬间：轻微镜头震动 + 音效。
- 全程适配移动端竖屏。

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| Gitee Pages 子目录路径 404 | `output:'export'` + 部署阶段再设 `basePath`/`assetPrefix`；`next dev` 本地预览验证 |
| Next.js 静态导出不支持浏览器 API（SSR 报错） | 游戏页 `'use client'`；Babylon/MediaPipe/Web Speech 仅在 `useEffect` 客户端加载 |
| Web Speech API 兼容性（Firefox/部分移动端不支持） | 检测 `'SpeechRecognition' in window`，不支持则隐藏语音按钮、保留文本输入 |
| MediaPipe 体积大 / 加载慢 | 按需动态 `import()`（仅 OfferingScene 引入）；CDN fallback |
| 摄像头权限被拒 / 无设备 | 降级为"点击掷杯"按钮，保证核心可玩 |
| 浏览器物理引擎性能（移动端） | 限制物理体数量（仅 2 片筊杯）、降级渲染、固定时间步长 |
| 物理落地不稳（站立/卡边） | 质心 + 底面形状调参；超时强制判定取主要朝向 |
| 免费音效 license 合规 | 仅采用 CC0/可商用素材，记录来源 |
| 文化准确性争议 | 标注"娱乐用途"、文案温和、不承诺灵验 |
