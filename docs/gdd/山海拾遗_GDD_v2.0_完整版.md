统一团队对以下问题的共识：

1. 用户玩完/玩一阵之后，**应该学到什么**；

## 2. 项目愿景与学习目标

### 2.1 一句话

《山海拾遗》是一款 **Web 端文化互动体验**：用户扮演「拾遗者」，在山海意境中探索、修复遗宝，**边动手边读懂**器物、传说、民俗与节气——在好玩里把传统文化「带回家」。

### 2.2 我们到底要用户带走什么

完成一段体验后，用户至少能做到下面之一（越多越好）：

| 层级 | 学习结果 | 例子 |
|------|----------|------|
| 认得 | 叫得出名字、分得清类型 | 「这是鼎，是青铜礼器」 |
| 懂得 | 知道一点来历或用途 | 「鼎曾是权力与祭祀的象征」 |
| 联得上 | 和今天的生活或节日有关 | 「端午赛龙舟、挂艾草从哪来」 |
| 愿意说 | 能向别人转述一句有趣的典故 | 分享卡上的一句「可讲出口的话」 |

### 2.3 学习范围（完整版覆盖）

1. **器物与工艺印象**：青铜、陶瓷、玉、书画、织绣等「长什么样、大概做什么用」  
2. **典籍与神话意象**：《山海经》等记载的神兽、地理、名物（标明再创作）  
3. **民俗与礼俗**：庙会、符箓文化符号、对联、灯谜等（健康、非迷信宣导）  
4. **岁时节日**：春节、元宵、清明、端午、七夕、中秋、重阳、冬至等  
5. **区域文化差异感**：中原礼乐、楚地浪漫、巴蜀遗珍、江南雅致、塞北丝路、仙山想象  

### 2.4 项目边界（已定案）

| 做 | 不做 |
|----|------|
| 打开网页即玩 | 强制注册登录 |
| 玩中读典故、看出处 | 试卷式刷题、说教弹窗轰炸 |
| 本地收藏与藏馆回顾 | 商城、内购、广告变现 |
| 策展式探索增加情境 | 重度数值养成、PVP、战力、战斗关卡 |
| 分享「我学到的一句」 | 好友系统、排行榜、登录墙 |
| 文化准确、可审校 | 伪称未授权「馆藏真品复刻售卖」 |
| 可动手的修复/解谜 | 伙伴升星、体力氪点、Roguelike 失败惩罚 |

### 2.5 目标用户

- 对传统文化好奇、但不想先读长论文的人  
- 学生、亲子、展览观众、站点随机访客  
- 海内外愿意用中/英了解中国文化的人  

**不是**：氪金网游用户、硬核竞技玩家。

### 2.6 对外定位句

> 在山海里修一件器物，就读懂一段中国故事。

### 2.7 体验关键词

`遇` · `修` · `读` · `藏` · `节` · `传`

（遇见 → 动手修复 → 阅读理解 → 收入藏馆 → 岁时共鸣 → 愿意讲述）

## 3. 体验支柱（玩中学）

### 3.1 四条支柱

| 优先级 | 支柱 | 含义 | 检验问题 |
|--------|------|------|----------|
| P0 | **学在动手里** | 知识点嵌在修复/解谜操作中，不只在通关后贴文 | 不做小游戏，还剩阅读吗？做了小游戏，是否学到了？ |
| P0 | **每件都有据** | 每张文化卡可查来源、可审校 | 顾问能否批注出处？ |
| P1 | **短时有收获** | 5–15 分钟能完成「修一件 + 读懂一句」 | 用户合上页面前是否多知道了一点？ |
| P1 | **藏是为了复习** | 藏馆不是炫耀墙，是「我的文化笔记本」 | 回看藏馆能否复习到知识点？ |

### 3.2 学习设计原则

1. **先体验，后解释**：先摸到纹样、听到钟声，再给名称与典故。  
2. **一次只带走一点**：每件文物主打 **1 个核心知识点 + 1 句可讲述**，避免百科倾倒。  
3. **区分「史证 / 民俗 / 经载」**：让用户知道什么是历史文物知识，什么是神话再创作。  
4. **谜题服务记忆**：对对子记的是意象，不是考倒用户；可提示、可重来。  
5. **零羞辱**：答错、修不好只降低完成评价，不嘲笑、不锁死内容。  

### 3.3 明确不做（v2.2 定案）

| 不做 | 原因 |
|------|------|
| 登录 / 商城 / 广告变现 | 公益零门槛 |
| **战斗 / 考验关卡 / 战力属性** | 抢注意力，不像文化展 |
| **伙伴养成、升星、编队** | 网游感；形象最多作图鉴插图 |
| **探索体力（足迹/履痕）墙** | 妨碍展览畅览与课堂使用 |
| 抽卡、通行证、日活压迫 | 非商业、非留存压榨产品 |
| 失败删进度、Roguelike 硬核死亡 | 与学习友好相悖 |

**完整版主系统只保留：** 文化地图遇见 → 修复/解谜 → 文化卡片 → 山海册/藏馆复习 → 岁时专题 → 分享。

## 4. 故事背景与文化地图

### 4.1 叙事设定（服务动机，不抢学习戏）

《山海经》里的奇珍与人间器物的灵，有的被时间磨淡了。  
用户是当代 **拾遗者**：不是盗宝，而是让名字、形制与故事重新被看见。  
**山海册** 是随身图鉴——记下修过的每一件，也记下读过的每一则。

叙事保持轻量：足够让人愿意点下一处；不做成必须看完才能玩的长剧。

### 4.2 文化区域（完整版）

| 区域 | 希望用户建立的文化印象 | 代表学习主题 |
|------|------------------------|--------------|
| 中原 | 礼乐与青铜时代的庄重 | 鼎、钟、纹样、祭祀与礼器概念 |
| 楚地 | 浪漫、巫祝与南方纹饰 | 凤鸟、漆器意象、楚辞风韵（适度） |
| 巴蜀 | 独特的古蜀遗珍感 | 纵目、黄金面具等**公开资料级**再阐释 |
| 江南 | 文人与瓷器、园林气韵 | 窑、绢本、雅集与物用 |
| 塞北 | 交流与路上的文明互鉴 | 马具、通商意象、胡汉交融的浅层认知 |
| 仙山 | 山海经地理与神兽想象 | 典籍记载 vs 艺术再创作 |
| 岁时幻境 | 节日活态文化 | 习俗、食物、传说、时序 |
| 海丝外篇 | 文化何以远行【设计假设】 | 传播与互鉴的浅层故事 |

### 4.3 术语（用户可见文案宜白话）

| 内部术语 | 用户可理解说法 | 含义 |
|----------|----------------|------|
| 山海遗宝 | 遗宝 / 文物卡片 | 可修复、可收藏的文化对象 |
| 阅历 | 我的学习进度 | 记录必学点与区域完成度（非体力） |
| 文化形象 | 插图里的神兽/意象 | 仅展示用，不养成、不出战 |
| 史证 / 民俗 / 经载 / 岁时 | 卡片角标 | 知识类型，避免混淆历史与神话 |

## 5. 整体体验结构

### 5.1 主路径（用户应感知到的故事）

```
打开网页 → 遇见一处山水/遗迹
    → 动手修复或解一道文化谜
    → 亮出文化卡片（名称·出处·一句故事·今日关联）
    → 收入我的藏馆（可回看、可布置、可分享）
    → 在节日专题里把知识放回「活的日子」
```

### 5.2 三层时间

| 层 | 时长 | 用户在做什么 | 学到什么 |
|----|------|--------------|----------|
| 一次修复 | 1–5 分钟 | 拼合、描摹、对诗、击节 | 形制/纹样/字句/节奏相关的一点知识 |
| 一程文化地图 | 8–20 分钟 | 在策展路线上点开 2–4 处展点 | 区域氛围 + 多张卡 |
| 回访 | 数日–数月 | 补图鉴、换展、过节专题 | 系统化、岁时化的文化图景 |

### 5.3 系统权重（v2.2）

| 系统 | 权重 | 职责 |
|------|------|------|
| 文化卡片 | **主** | 知识落点与出处 |
| 修复 / 解谜 | **主** | 动手记住形制与典故线索 |
| 藏馆 / 山海册 | **高** | 复习、策展、展览感 |
| 文化地图 / 见闻 | 中高 | 情境化遇见，避免纯列表 |
| 岁时节日 | 中 | 活态文化 |
| 文化形象插图 | 低 | 仅点缀记忆，不养成 |
| 战斗 / 体力 / 养成 | **无** | 已删除 |

### 5.4 单次访问的理想收获

- **最短路径（3–5 分钟）**：今日一器或任选 1 件 → 修 → 读完 → 可选分享。  
- **标准路径（15 分钟）**：沿一条文化地图路线看 2–3 处 → 回馆复习。  
- **沉浸路径（30 分钟）**：一区域必学清单走完 + 主题展布置 + 节日页。  

### 5.5 双入口（展览友好）【设计假设】

| 入口 | 说明 |
|------|------|
| **展厅畅览** | 默认。从图鉴/区域列表直接选器物学习，无体力、无战斗。 |
| **山海漫步** | 可选。在手卷地图上按策展路线点展点，增加情境，仍无失败惩罚。 |

## 6. 文化地图：在情境里遇见（非 Roguelike 闯关）

### 6.1 为什么要有地图

若只有图鉴列表，容易变成「点开百科」。  
**策展式文化地图** 提供情境记忆：在楚地遇见凤纹、在端午听到鼓点，比在菜单里点分类名更好记。  
它是 **数字展陈动线**，不是刷层爬塔。

### 6.2 形态（v2.2 定案）

- 山水手卷上的 **固定/半固定展点**（亭、渡、庙、书院、窑址、关隘等）。  
- **无行动力消耗**；想看几处看几处，随时退出，进度保留。  
- 每区提供 1–2 条 **推荐观展路线**（必学点按序点亮）；也允许自由点选。  
- 可有轻度「今日路线」随机排序，但 **知识内容固定可查**，重要条目必在山海册可直达。  
- **不做**：程序生成刷局、精英怪、死亡结算、层数难度爬升。  

### 6.3 展点类型（均应对文化有贡献）

| 展点 | 用户行为 | 学习贡献 |
|------|----------|----------|
| 遗宝台 | 修复 / 解谜 | 主学习 |
| 书院 | 诗句、典故轻互动 | 文学与典故 |
| 庙会角 | 民俗小互动 | 活态民俗 |
| 说明牌 | 短文见闻（只读） | 区域背景 |
| 小结亭 | 区域 3 句回顾 | 结构化复习 |

**删除展点类型：考验 / 战斗 / 补给买活。**

### 6.4 编排原则（服务学习）

1. 推荐路线上 **必学点全覆盖**，不靠运气。  
2. 说明牌与遗宝台交替，有读有做。  
3. 区域主题统一，保证「这片地有这片地的文化味」。  
4. 山海册始终可直达任意已解锁/可预览条目，避免「地图里找不到就学不到」。  

### 6.5 提示强度（替代「游戏难度」）

- **导览**：提示多、吸附强，适合展览与课堂。  
- **细读**：标准提示。  
- **自测**：少提示（仅影响修复评价印记，**不锁知识阅读**；仍可点「先阅读」）。

## 7. 拾遗与文化卡片（知识本体）

> 文化卡片是本项目的 **知识数据库面向用户的样子**。玩法可以变，卡片结构应稳定。

### 7.1 一张合格的文化卡片必须有

| 字段 | 要求 | 学习作用 |
|------|------|----------|
| 名称 | 准确、可检索 | 认得 |
| 类型角标 | 史证 / 民俗 / 经载 / 岁时 | 判断知识性质 |
| 一句话介绍 | ≤ 30 字 | 3 秒能懂 |
| 核心知识点 | 1 条，80–150 字 | 懂得 |
| 出处 / 依据 | 典籍、公开资料、民俗志等 | 可追溯 |
| 今日关联 | 1 句，连到生活或节日 | 联得上 |
| 可讲述句 | 1 句，适合分享出口 | 愿意说 |
| 延伸（可选） | 折叠的 1–2 段 | 想深读的人 |

**【设计假设】** 默认展开：名称 + 一句话 + 核心知识点 + 今日关联；出处与延伸可点开。

### 7.2 分类

**品级**（表示展示精致度与修复阶段数，**不是氪金等级话术**）：

| 标记 | 含义 | 完整版数量感 |
|------|------|----------------|
| 寻常 | 易遇、一阶修复 | 约 70 |
| 雅器 | 常见主题物 | 约 65 |
| 礼器 | 区域代表 | 约 45 |
| 珍遗 | 多阶段修复 + 更完整故事 | 约 30 |
| 名物 | 招牌内容 + 专题演出 | 约 10 |
| **合计** | | **约 220** |

**物类**：青铜 · 陶瓷 · 玉石 · 书画 · 织物 · 礼乐器 · 民俗物件 · 符箓意象（文化符号向） · 异兽名物（经载再创作） · 节气灵物  

### 7.3 获取与重复

- 主来源：探索遗迹、章节目标、节日专题。  
- 重复遇到：可转为「拓片 / 笔记残页」用于解锁卡片皮肤态或藏馆小装饰（**无商城**），并提示：「再读一遍核心知识点」。  

### 7.4 图鉴（山海册）

- 按区域、物类、节日、系列（如「认识礼器」「节气物语」）浏览。  
- 系列完成时给 **短小结**（5 句话复习），比给「抽奖」更符合学习目标。  
- 进度可视化：为的是「我还没见过什么」，不是制造焦虑。  

### 7.5 内容生产规范（完整版产能关键）

每写一件必须过：

1. 核心知识点是否 **唯一且好记**？  
2. 出处是否可查？史证是否表述谨慎？  
3. 修复玩法是否帮助记住该点（例如铭文、纹样、器形）？  
4. 可讲述句是否适合口口相传？  
5. 是否避免地域歧视、封建迷信宣导、不当戏说？

## 8. 修复与解谜（主玩法 = 主学习）

### 8.1 为什么「修」是主指纹

- 手一动，注意力就在器物上；  
- 纹样、器形、字句会在操作里被多看几眼；  
- 完成后的「亮相」天然适合接文化卡片。  

**学是目的，修是记忆编码方式。**

### 8.2 通用评价（反馈学习投入，而非羞辱）

完整度 · 准确度 · 是否借助提示 · 是否从容完成  

评价用于：

- 鼓励再试一次（想拿更好的完成表现）；  
- 决定卡片上是否点亮「完璧」小印；  
- **不**用于贩卖复活或锁内容。  

### 8.3 玩法模板与学习映射

| 模板 | 操作 | 主要学什么 |
|------|------|------------|
| T1 器形拼合 | 拼回鼎、壶、陶 | 器形、部件、用途线索 |
| T2 纹样补笔 | 沿痕描云雷、凤鸟等 | 纹样名称与区域风格 |
| T3 符图式连线 | 按图连线、方位 | 传统符号与图式结构（文化知识，非宗教修炼教学） |
| T4 诗句与对仗 | 填字、上下联 | 意象、节日诗、基础对仗趣味 |
| T5 节奏击节 | 鼓点、编钟序 | 礼乐/民俗节奏、端午鼓等 |
| T6 方位与序 | 九宫、简单卦序游戏化 | 「古典知识游戏」层，表述克制 |
| T7 综合修复台 | 多阶段 | 名物级完整故事 |
| T8 节日专题 | 包粽步骤、灯谜等 | 节日习俗流程 |

### 8.4 学习增强设计（每关建议具备）

1. **修前一眼**：破损态上有淡淡名称或类型提示。  
2. **修中提示**：册灵可指出「这道纹叫什么」（可选，不强制）。  
3. **修后必读**：卡片主知识点不可被跳过到消失——可点「知道了」，但要曝光一次。【设计假设】  
4. **错题温柔**：诗句类给韵部/意象提示，而不是只说「错误」。  

### 8.5 招牌时刻（传播 + 记忆）

完成度较高时：

1. 裂痕合拢 / 墨色补全；  
2. 名称印章落下；  
3. **可讲述句** 出现；  
4. 引导「读完小知识」；  
5. 可生成分享卡。  

### 8.6 无障碍与包容

- 可放慢、可加强吸附、可更多提示；  
- 色弱友好；  
- 减少闪烁；  
- 允许跳过操作进入阅读（标记为「先阅读」），保证 **只想学的人也能学到**。【设计假设】

## 9. 文化形象（仅展示，不养成）

### 9.1 定位（v2.2 定案）

部分经载神兽、节气拟人、器物意象，可以以 **插图 / 立绘** 出现在：

- 文化卡片页旁；  
- 区域枢纽氛围；  
- 藏馆装饰位。  

用途只有：**让知识更好记、让展览更有温度**。

### 9.2 明确不做

- 编队、出战、技能、等级、升星、羁绊数值  
- 与形象绑定的战斗或考验关卡  
- 抽取或重复碎片合成形象  

### 9.3 数量

- 不设「90 可养成伙伴」指标。  
- 完整版按内容需要配置 **装饰立绘** 即可（例如每区 2–4 张氛围形象 + 少量名物相关插图）。  
- 工期紧时 **可以不做任何形象**，只保留器物卡与修复，不影响文化主目标。

## 10. 山海藏馆（复习与展示）

### 10.1 定位

- **复习本**：回看修过的器物与知识点；  
- **小展览**：自己排一排主题（礼器墙、端午角）；  
- **讲述起点**：一键生成「我的展览」分享图。  

### 10.2 空间（概念）

门厅（今日一器）· 主展厅 · 主题侧厅 · 修复案头（重读/再修）· 书阁（山海册）  
（无「出战灵圃」；若有文化形象，仅作展陈插图。）

### 10.3 策展学习玩法

- 按系列推荐陈列：「把中原礼器放在一起，再读一次区域小结」。  
- 布置完成可点亮 **主题理解小印**（表示你复习过，不是付费勋章）。  

### 10.4 研究

- 用重复拓片解锁：更清晰的器形展示、旋转/视差、语音朗读核心知识点。  
- 目的是 **加深印象**，不是养成肝度。

## 11. 节日与岁时专题

### 11.1 目的

把文物知识接回 **中国人的时间**：过节时来，带着玩，也带着懂。

### 11.2 完整版节日框架

| 节日 | 玩中学重点 |
|------|------------|
| 春节 | 门神意象、春联对仗趣味、年俗健康叙事 |
| 元宵 | 灯谜、灯彩 |
| 清明 | 踏青时序、克制庄重的民俗说明 |
| 端午 | 龙舟鼓点、艾草与粽的习俗、相关文学记忆 |
| 七夕 | 星象传说与诗歌意象（再创作边界清晰） |
| 中秋 | 月与团圆、相关器物与诗 |

### 12.1 记录什么

全部 **本地存储**；可选导出 JSON 备份。

### 12.2 还要不要「资源」？

**v2.2 默认：几乎不需要资源货币。**

| 可选轻量标记 | 用途 |
|--------------|------|
| 拓片 / 笔记 | 仅表示「又读过一次」，用于点亮展示增强（朗读、高清器形），**不设稀缺肝度** |

- 任意未修之物：允许 **先阅读卡片再动手**（或只阅读）。  
- 重要学习内容在山海册中始终可追踪，不依赖地图随机。

## 14. 引导与学习节奏

### 14.1 前 10 分钟（幸福路径）

| 回访 3 | 能讲 1 个节日或 1 则简短典故 |

### 14.3 系统展开顺序（示例）

先：修复、文化卡、藏馆、山海册

## 19. 本地化、合规与文化审校

### 19.1 本地化

- 知识文案全翻译；纹样与器物图通用。  
- 强依赖汉字的谜题：英文轨提供 **等价认知题**（认纹样、排顺序、选用途）。  

### 19.2 文化审校流程

1. 每条史证写清依据。  
2. 双人校对 + 顾问抽检。  
3. 争议表述降级为「经载再创作」或改写。  
4. 可热修静态文案。  

### 19.3 合规

- 关于页：公益目的、资料说明、免责声明。  
- 神话与民俗：尊重公序良俗，避免迷信实操教学。  
- 隐私：本地进度说明；统计可选且最小化。  
- 知识产权：字体、音乐、参考图合法来源。

## 22. 风险与开放问题

| 风险 | 缓解 |
|------|------|
| 好玩但学不到 | 修后必读核心点；模板映射知识点；验收用「复述测试」 |
| 学到了但不好玩 | Phase0 先打磨手感；提示与吸附 |
| 百科过载 | 每卡 1 个核心点；延伸折叠 |
| 史实错误 | 审校流程与角标 |
| 被当成网游 | 关于页定性；无战斗/养成/体力；文案去网游语感 |
| 内容产能不足 | 工具化关卡；先保必学清单再铺数量 |
| 只有阅读不够吸引 | 保留修复手感作为展陈互动，而非加战斗 |

1. ~~轻考验/战斗~~ → **不做**

1. 英服谜题并联策略细则；

### 附录 B · 文化卡片样例

**名称**：云雷纹铜鼎（史证灵感 · 再创作展示）  
**一句话**：青铜礼器，腹上云雷纹环绕。  
**核心知识点**：鼎在古代常与祭祀、礼制相关；云雷纹是商周青铜器上常见的几何纹样，像云与雷的连续回旋。  
**出处**：公开博物馆图录与通识读物中的器形与纹样说明（正式稿写具体可核验来源）。  
**今日关联**：今天我们在博物馆里见到鼎，多把它当作理解古代礼制与工匠传统的一把钥匙。  
**可讲述句**：修这只鼎时，我认出了肚子上转圈的云雷纹。  
**修复**：T1 器形拼合 → 纹样接缝。
