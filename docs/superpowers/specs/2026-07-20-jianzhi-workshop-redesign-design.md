## 2. 体验骨架

### 2.1 身份与入口

- 玩家是**工坊学徒**，不是博物馆游客，也不是纯自由画布用户。
- 默认唯一主 CTA：**进入工坊**。
- 次要入口：自习台、师傅纹样册、作品墙（进度/设置收入次级）。
- 去掉「6 个平权菜单项」的应用壳感；去掉「REX GAME / 非遗互动展品 03 / CHINESE PAPER-CUTTING / JIANZHI」式双语展馆编号作为主视觉。

### 2.2 双阶段进度

```
阶段一 · 基础功课（线性）
  课 1 入门 → 课 2 → … → 课 N 出师
        │
        └─ 出师后解锁 ──► 阶段二 · 时令委托（开放）
                              春节 / 婚庆 / 中秋 …
```

- **阶段一**：师傅派功课。必须完成当前课才能开下一课。
- **阶段二**：完成出师课后解锁时令委托。委托可并行，奖励偏图谱与荣誉，不堵主路径。
- **自习台**：任意时刻可进（建议课 1 完成后强调引导）；自由折剪、拼吉语、存作品；不推进功课进度。

### 2.3 情感弧线

学徒从「第一次展开见福」→「读懂谐音拼句」→「时令场景下用吉语」→「能独立完成委托」→ 出师感。

## 3. 视觉系统（材质优先）

### 3.1 原则

1. **材质 > 装饰**：红纸纤维、刀口锯齿、镂空透光、折痕阴影优先于插画海报、超大装饰字、实心偏移阴影。
2. **年俗克制**：朱红是纸与印的颜色，不是整页喊「过年」。
3. **站点同源、不克隆**：使用 `--font-display` / `--font-body`、克制金箔描线、朱砂强调；**不做**整页漆黑褐底（与圣杯首页区分，保留纸工作坊气质）。
4. **去西式版画壳**：禁止主视觉 Georgia 序号、英文巨型水印、硬直角 neo-brutalist 按钮作为默认语言。

### 3.2 色与表面

| Token 角色 | 建议 | 用途 |
|------------|------|------|
| 工坊底 | 暖木案 / 旧纸米 `#E8DCC0`～`#DFD0AF` 一带 | 页面底，可有极轻木纹或颗粒 |
| 红纸 | 祭红/朱砂 `#9F251D`～`#A6332B` | 画布纸面、主 CTA |
| 墨 | `#1A120B` / `#2A1B12` | 正文、描边 |
| 金箔点缀 | `var(--c-gold)` 系 | 印章边、焦点、完成态，面积克制 |
| 镂空透光 | 画布背后偏亮或微窗光 | 展开后镂空「透」出来 |

实现上：`JianzhiGame.module.css` 局部变量对齐上述角色，能复用 globals 的则引用 CSS 变量，不复制第三套无关色板。

### 3.3 组件语言

| 元件 | 新语言 | 禁止 |
|------|--------|------|
| 主按钮 | 圆角或微圆（对齐站点 `--r` / pill 思路）、朱红底、轻投影 | 6px 实心黑偏移、硬直角默认 |
| 卡片/读帖 | 宣纸面 + 细墨线/金线 + 可选朱印角标 | 厚黑 9px 分割作唯一结构 |
| 导航条 | 工坊顶栏：返回 · 课名 · 折法状态 | 应用式 01–06 列表 |
| 舞台 | 红纸居中，四周暗一档木案，聚焦「这一张纸」 | 全屏浅 kraft 无层次 |
| 印章 | 小朱文方印作完成/认证 | 大号装饰印抢戏 |

### 3.4 材质实现要求（画布 + UI）

**画布（`paperCanvas` 增强，可分期）：**

- 红纸表面：细噪点 / 纤维感（canvas 或 CSS），避免纯 flat `#ff0000`。
- 镂空：destination-out 后能感到「洞」——边缘微不齐或内阴影暗示厚度。
- 折叠态：可见折痕与折半阴影。
- 展开动画：时间略拉长（约 0.8–1.2s，尊重 reducedMotion），强调「一纸成花」的揭晓。

**UI：**

- 纹样缩略图（`MotifGlyph`）继续红纸底 + 剪影，与大画布同一材质语言。
- 贴窗花结算：可选；有则做成**半透明窗纸 + 镂空投影**的材质演示，不做主题乐园贴纸墙。若工期紧，一期可用「裱在纸托上的作品卡」替代。

### 3.5 字体与文案语气

- 标题：`var(--font-display)`（宋体展示）。
- 正文：`var(--font-body)`。
- 师傅/纸灵台词：短、口语、像带徒，避免长百科段落堆在主路径。
- 百科长文：放师傅纹样册 / 读帖折叠「细读」。

## 4. 玩法循环

### 4.1 单课五段（核心）

```
① 拜帖（读帖）     30–60s
     源流一句话 + 本课技法 + 本课要拼的吉语/目标
     可跳过「细读」，不可跳过目标确认
        ↓
② 折纸              强调折法与文化绑定
     默认本课 foldSuggestion；首次接触该折法时短提示
        ↓
③ 落剪 / 戳纹样      主操作
     纹样戳印为主；自由剪为辅（不挡过关）
     目标纹样在侧栏 checklist；谐音关系在旁白或目标区点明
        ↓
④ 展开揭晓          仪式高潮
     动画 + 音效
     若拼成目标吉语：全屏/半屏揭示「莲+鱼 → 连年有余」及一句文化句
     未达标：温和提示缺哪些纹样，回 ③
        ↓
⑤ 巩固 + 结算        轻量
     优先：短理解题（单选，现有 quiz 可复用结构）
     然后：解锁档案条目、新纹样、进度
     可选：存作品 / 下载 / 贴窗花展示
```

**设计意图**：文化不靠侧栏自学，而嵌在 ① 目标、③ 旁白、④ 揭晓三处；④ 是「看图说吉话」的顿悟点。

### 4.2 过关条件

- 放置本课 `objectiveMotifIds`（或目标 combo 所需 motif 全集）即可过展开检定。
- 理解题：选错给解释并可再选；**不**因答错锁课（学徒友好）。一期可保持「必须点选才能结算」。
- 委托关：同结构，目标改为委托 brief（如「剪一副春节窗花：福 + 鱼」）。

### 4.3 自习台

- 无目标 checklist（或仅显示「本次已拼成的吉语」）。
- 折法自由切换；拼出库内 combo 时 toast + 写入 discoveredCombos。
- 存作品 / 下载保留。

### 4.4 时令委托（阶段二）

- 卡片：时令名 · 场景一句 · 目标纹样/吉语 · 推荐折法 · 完成印记。
- 完成后：解锁对应文化档案或 combo 注解强化，不强制线性。
- 一期最少 2 个委托（春节、婚庆）；中秋等可二期加。

### 4.5 音效与动效

- 保留 `createPaperAudio`：落剪、展开铃。
- 展开与成句用更明确的层级反馈（toast 不足以承载顿悟时用揭晓层）。
- 全局遵守 `reducedMotion` / `muted`。

## 7. 验收标准

### 7.1 视觉

- [ ] 剪纸页主视觉无 Georgia 巨型英文水印、无默认实心偏移黑阴影按钮语言。
- [ ] 舞台中心红纸可读为「一张纸」；有折痕/展开/镂空层次（至少展开仪式明确）。
- [ ] 字体与站点 display/body 一致；金箔仅点缀。
- [ ] 移动端功课地图与工坊可完成主路径（不依赖桌面-only 布局）。

### 7.2 流程

- [ ] 默认进入「功课地图」，主 CTA 为当前课「上工」。
- [ ] 单课固定顺序：拜帖 → 折 → 剪 → 展开揭晓 → 巩固结算。
- [ ] 线性锁课正确；出师后出现委托区（若一期含委托）。
- [ ] 自习台不错误推进功课进度。
- [ ] 成句揭晓明确展示「纹样 → 吉语」映射，而非仅 toast。

### 7.3 文化

- [ ] 每课读帖含源流/技法/本课焦点；目标吉语的谐音或象征在揭晓层复现。
- [ ] 文化档案仍随进度解锁，证据标签保留。
- [ ] 主路径文案为带徒语气，百科细节在图鉴细读。

### 7.4 技术

- [ ] 进度 v2 写入新 key；v1 可迁移或安全重置。
- [ ] 现有纹样 draw 与 combo 检测无回归（单元/手工：莲鱼、蝠桃等）。
- [ ] `reducedMotion` / `muted` 仍生效。
- [ ] 静态导出（Gitee Pages）路径与 SEO 元数据仍有效。

## 8. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 一体重做范围过大 | 严格分期；一期必须出师可玩，委托可标「即将开放」但结构预留 |
| 材质做不真 | 先做展开揭晓层 + 纸面噪点 + 侧光，避免大改物理 |
| 内容重写拖期 | 功课表先绑现有 motif/combo id，只改文案与顺序 |
| 进度丢失客诉 | v2 迁移 + 首次进入说明 |
| 与英歌版画风不一致 | 接受：剪纸走材质工坊，英歌可另轨；全站以 token 同源而非皮肤克隆 |

---

## 9. 成功定义（一句话）

玩家进入后感到自己是**在红纸上跟师傅学一门手艺**；每完成一课都经历一次**展开见花、读懂吉语**的顿悟；视觉是**纸与刀的质感**，而不是另一套版画网站皮肤。

## 10. 实现入口（批准 spec 后）

1. 调用 `writing-plans` 产出分步实现计划（建议按：types/progress → lessons 内容 → 视图状态机 → CSS 材质 → 揭晓层 → 委托/迁移）。
2. 实现前走 `pre-edit-safety-gate`。
3. 完成前走 `verification-before-completion`。
