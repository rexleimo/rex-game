# Spec 3 · 纸上生花「纸上剧场」重生

- 日期:2026-07-20
- 状态:已批准(brainstorm 逐项确认 + 画板 mockup 通过),直接实施
- 前置:Spec 1(设计系统 v2)、Spec 2(文化页矩阵)已交付

## 1. 背景

剪纸游戏(纸上生花)是用户指定的**重点优化项**:玩法闭环完整(7 课、4 折法、14 纹样、9 吉语、图鉴/作品/委托,约 2700 行),但视觉执行朴素、体验节奏平铺、文化寓含停留在文本层。方向 C 的设计系统已就绪,现在把它重铸为「纸上剧场」。

## 2. 决策记录(本轮 brainstorm 确认)

| 决策点 | 结论 |
|---|---|
| 剧场结构 | **剧场主线 + 工坊保留**:4 幕场景为主线;自习台改「练功房」保留;图鉴/作品墙重铸保留 |
| 幕的划分 | **4 幕 + 纸灵贯穿报幕**:幕1 年夜·窗花(对折基础)→ 幕2 新婚·双喜(对称进阶)→ 幕3 端午·五毒(自由剪)→ 幕4 团圆·团花(团花+吉语大考);每幕一个场景角色讲文化台词 |
| 体验增强 | 三项全做:**展开电影时刻**(全屏慢镜+金光粒子+吉语浮现)、**作品分享卡**(暗场金框+吉语+水印,现有下载升级)、**民艺馆作品墙**(金框装裱+射灯) |
| 美术路线 | AI 生成场景大图(4 幕氛围)+ 纸纹理;UI/纹样用 SVG;失败回落 SVG 剪影场景 |
| 引擎边界 | `runtime/`(paperCanvas/fold/audio)与 `content/` 数据结构(motifs/combos/lessons)**复用不动**;重构 UI/体验层 |
| 视觉语言 | 方向 C 设计系统(tokens.css/gallery.css `--g-*`),游戏内复用 token,不新建色系 |

## 3. 范围

**做:**
1. 剧场框架:幕布地图(主页)+ 幕进度 + 解锁/揭幕机制
2. 幕内创作界面:场景区 + 剪纸台 + 纹样/目标区 + 角色台词条
3. 4 幕内容与课程映射(7 课重排入幕,含台词)
4. 展开电影时刻(unfold ceremony)
5. 作品分享卡生成(canvas → 品牌卡 PNG)
6. 民艺馆作品墙(主页或独立视图)
7. AI 场景大图 ×4 + 纸纹理 ×1
8. localStorage 旧进度迁移(已解锁纹样/吉语不清零)

**不做:** 引擎算法改动(fold/paperCanvas 内部)、新纹样/新吉语(沿用 14+9)、游戏页 SEO 文案结构(spec 2 模式沿用)、圣杯/英歌(spec 4)。

**成功标准:** 4 幕完整可通关;展开电影时刻在每课末呈现;作品可生成分享卡并下载;作品墙装裱展示;旧玩家进度平滑迁移;全部契约测试通过;构建无破版。

## 4. 剧场设计

### 4.1 幕布地图(游戏主页)

- 四张门帘卡片:红幕=当前幕、暗金幕=已解锁可回顾、灰幕=未解锁(🔒)
- 每张:幕序号(第一幕)/ 主题(年夜 · 窗花)/ 状态(进行中/已点亮/未解锁)
- 进入幕=「揭幕」动画(幕布向两侧拉开,CSS transform)
- 主页同时承载:民艺馆作品墙入口、图鉴入口、练功房入口、设置

### 4.2 幕内创作界面(最高频屏)

- **顶栏**:幕序号+主题 / 课进度点(本幕 2-3 课)/ 图鉴·作品·设置
- **场景区**:AI 场景大图(年夜雪窗/婚房红烛/端午艾草/团圆灯下),角标场景题字;随幕切换
- **剪纸台**:折面(canvas,引擎渲染)+ 剪刀光标 + 剪口虚线;纹样戳印格(本课可用纹样)+ 自由剪切换
- **目标区**:本课目标(如「剪一枚福字纹,贴上窗棂迎年」)+「展开」主按钮(朱砂)/ 重折 / 求助纸灵
- **台词条**:角色头像 + 逐句推进台词;每课开场 2-4 句(一句引导操作、一句文化点),完成后再 1-2 句点评;点击继续,可跳过

### 4.3 课程映射(7 课 → 4 幕,按实际课程内容校准)

- 幕1 年夜·窗花:L1 awaken(单面起剪·福)、L2 symmetry(对折·鱼)(角色:奶奶;文化点:连住不断=福气不断、连年有余)
- 幕2 新婚·双喜:L3 rebus-intro(莲鱼=连年有余)、L4 window-four(梅)(角色:喜娘;文化点:双喜、喜上眉梢、并蒂莲)
- 幕3 丝路·团花:L5 silk-rosette(团花·蝶恋花)(角色:丝路老艺人;文化点:团花与丝路传播)
- 幕4 福寿·团圆:L6 fu-shou(蝠桃=福寿双全)、L7 graduate(吉语大考)(角色:纸灵亲授;文化点:看图说吉话)
- 校准说明:初稿幕3 为「端午·五毒(自由剪)」,但现有 L5 实为团花丝路,故改主题贴合内容(用户确认);端午主题后续以时令委托形式加入(委托系统现成)
- 台词写作:口语、短句、每句 ≤40 字;文化点遵循三分表述(常见说法/地区差异/游戏设计);角色不说教,说「家常话里的讲究」;**课开场台词优先复用 lessons[].narrative 既有文案**

### 4.4 解锁与点亮

- 初始仅幕1 解锁;完成一幕全部课 → 该幕「点亮」(场景区亮起+烟花粒子),下一幕解锁
- 练功房随时可用:自由折剪(全折法+已解锁纹样),不进剧场主线
- 图鉴/吉语/委托(出师后时令委托)保留,入口收进主页

## 5. 体验增强

### 5.1 展开电影时刻(UnfoldCeremony)

- 触发:点击「展开」且目标达成 → 全屏遮罩仪式:折面慢镜展开(1.2s)→ 金光粒子 + 纸屑飞散 → 完整窗花定格 → 吉语浮现(若拼出组合)
- 配乐/音效:沿用 runtime/audio.ts,展开音 + 新增「定格」和声音(可 WebAudio 合成,不引音频文件)
- 跳过:点击可跳;`prefers-reduced-motion` 下直接定格展示

### 5.2 作品分享卡(ShareCard)

- 每幅完成品 → 一键生成 1080×1350(4:5 社交卡)PNG:暗场底 + 金框装裱窗花(canvas 重绘)+ 吉语/落款 + 「REX-GAME · 纸上生花」水印
- 实现:离屏 canvas 合成 → `canvas.toBlob` → 下载(静态站点无分享 API 依赖)
- 入口:展开仪式定格帧 + 作品墙每幅作品

### 5.3 民艺馆作品墙(GalleryWall)

- 主页区块:作品按时间倒序,金框+射灯(CSS cone 光)+ 吉语标签;空态引导(「剪完第一课,这里会亮起第一盏灯」)
- 每幅:点击查看大图 + 分享卡下载

## 6. 美术资产

- AI 生成(统一提示词:暗场、暖金光源、剪纸剧场、民俗场景、无文字):`scene-newyear.png`(年夜雪窗)/ `scene-wedding.png`(婚房红烛)/ `scene-duanwu.png`(端午艾草)/ `scene-reunion.png`(团圆灯下),webp 压缩入 `public/assets/jianzhi/scenes/`
- 纸纹理 ×1(洒金红纸,平铺):`paper-red.webp`,剪纸台/卡片低透明叠加
- 失败回落:SVG 剪影场景(红金渐变+几何窗棂),保证无图也能上线

- 主组件重构:`JianzhiGame.tsx`(1306 行)拆分为状态机 hook(`useTheater.ts`:幕/课/进度)+ 视图组件(TheaterMap/ActStage/CutBench/DialogueBar/UnfoldCeremony/GalleryWall/ShareCard),保留于 `src/games/jianzhi/`
- 样式:弃用 `JianzhiGame.module.css` 大部,新建 `theater.css`(全局,引用 `--g-*` token);游戏页壳继续独立全屏
- 进度迁移:读取旧 localStorage key,映射到新 schema(幕/课进度);迁移代码含测试
