# Spec 1 · 设计系统 v2「数字民艺」+ 首页重设计

- 日期:2026-07-20
- 状态:已批准(brainstorm 全程逐项确认),待写实现计划
- 作者:rex + Claude(brainstorming 流程产出)

---

## 1. 背景与问题

rex-game 目前给人「廉价感」,具体表现为:

1. **UI/UX 廉价感** — 站点门面(首页)是浅色博物馆主题 + bento 卡片,缺少视觉冲击力和排版工艺;站点 title「趣玩民俗小游戏站」本身就有廉价感。
2. **宣传效果不足** — 首页没有把站点作为「产品」卖出去;社交流量几乎为零。
3. **SEO 不到位** — 内容深度/数量不足、GEO/AI 搜索未覆盖、传统排名没起来、社交分享无钩子。(技术层 metadata/JSON-LD/sitemap 已完整,不是短板。)
4. **文化寓含浅** — 文化内容停留在「轻量导读」级别。
5. **剪纸游戏简陋** — 玩法闭环完整(约 2700 行,功课/折剪/图鉴/收藏),但视觉执行和体验节奏朴素,是重点优化项。

## 2. 总决策记录(brainstorm 确认,跨子项目)

| 决策点 | 结论 |
|---|---|
| 推进方式 | 设计系统先行,再逐个应用;拆为多个子项目,各自走 spec → 计划 → 实现 |
| 设计方向 | **C · 当代数字艺术**(参照 teamLab / UCCA 数字展):深漆底 + 高饱和朱红 + 超大衬线字 + 漂浮剪纸装置 + 强动效 |
| 文化页形态 | **明暗双轨**:暗场展厅式页头 + 宣纸米白阅读正文(spec 2 实现) |
| 剪纸重构幅度 | 全量视觉重铸 + 概念可重生(不设限) |
| 剪纸概念 | **纸上剧场(叙事关)**:每课一个民俗场景(年夜贴窗花/新婚剪双喜/端午剪五毒),在场景中剪、剪完场景点亮,文化寓意由角色之口讲出(spec 3 实现) |
| SEO 目标 | 四项全要:内容矩阵深化、GEO/AI 覆盖、传统排名、社交流量 |
| 美术路线 | **AI 生成大图 + SVG UI/纹样**;纸张纹理等材质用 AI,UI 与剪纸符号用 SVG |
| 首屏主张 | **「一座可以玩的中国民艺馆」** |
| 站点 title | 「rex-game · 可玩的民俗文化馆」(弃用「趣玩民俗小游戏站」) |

## 6. 设计系统 v2 tokens

### 6.1 色彩(深漆底 × 朱 × 金)

| Token | 值 | 用途 |
|---|---|---|
| `--g-abyss` | `#0A0705` | 页面最深底 |
| `--g-lacquer` | `#120B07` | 主底色(漆底) |
| `--g-float` | `#1A130C` | 浮面(卡片、区块) |
| `--g-card` | `#241A10` | 更高浮层 |
| `--g-cinnabar-hi` | `#E8452F` | 高光朱:大字点缀、hover |
| `--g-cinnabar` | `#C82E21` | 正朱:主 CTA、装置 |
| `--g-cinnabar-deep` | `#A81E14` | 深朱:压暗、渐变底 |
| `--g-gold` | `#C9A24B` | 古铜金:线条、标签、编号 |
| `--g-gold-hi` | `#E8CF9A` | 高光金:仅渐变高光 |
| `--g-paper` | `#F7F1E6` | 宣纸:反白面、印章底 |
| `--g-cream` | `#E8DCC4` | 暖米白:暗底正文 |
| 文本阶梯 | `#F5EDE0` 标题 / `rgba(232,220,196,.68)` 正文 / `rgba(232,220,196,.45)` 弱化 | — |

不引入蓝绿等功能色;成功/信息态用金/朱表达。

### 6.2 字体排印

- **展示字**:Noto Serif SC,600–700;三档 display — `clamp(3rem,8vw,6rem)` / `clamp(2.5rem,6vw,4.5rem)` / `clamp(2rem,5vw,3.5rem)`,行高 1.1–1.15,字距 .02–.08em
- **章节标题**:Noto Serif SC,`clamp(1.75rem,4vw,2.75rem)`
- **正文**:Noto Sans SC,16–17px,行高 1.8–1.9(中文长文)
- **标签/英文**:11–12px,大写,letter-spacing .2–.4em(展览标签感)
- **竖排**(`writing-mode: vertical-lr`)仅作点缀:卷首语、引文
- 编号用中文数字(〇一/壹)与「No.01」双语混排

### 6.3 间距

- 8px 基栅;章节间距 `clamp(7.5rem,15vw,12.5rem)`(120–200px)
- **大留白是高级感的第一来源**,宁可删元素不可挤

### 6.4 材质语言

1. **洒金宣纸纹理** — AI 生成,平铺,叠加透明度 ≤8%
2. **金箔渐变标题** — `linear-gradient(120deg,#F5EDE0 30%,#C9A24B 70%)` + `background-clip: text`
3. **剪纸镂空装置** — SVG(红/金),漂浮 + 滚动视差
4. **朱红光晕** — `radial-gradient` 大范围低透明,造暗场纵深

### 6.5 动效规范

- 时长三档:200ms(微交互)/ 500ms(进场)/ 900ms(hero)
- 缓动:`cubic-bezier(.22,1,.36,1)`(expo-out 系)
- 滚动触发:IntersectionObserver + Reveal 组件;CSS scroll-timeline 做 `@supports` 渐进增强,**不引入滚动动画库**
- 漂浮装置:CSS keyframes + `transform: translate3d`,每屏 ≤6 个
- `prefers-reduced-motion`:全部降级为透明度渐变

## 7. 首页信息架构(最终文案)

### S1 · 首屏装置 Hero(全屏暗场)

- 导航:左 `REX-GAME` 字标,右 展品/文化馆/关于
- eyebrow:`可玩的民俗文化馆 · INTERACTIVE FOLK MUSEUM`
- H1:**一座可以玩的 / 中国民艺馆**(「民艺馆」用高光朱)
- lead:`掷筊问愿、英歌合槌、折剪生花——三件民俗展品,无需下载,即开即玩;玩完再读背后的称法、节奏与吉语。`
- CTA:主「进入展厅」(朱砂)/ 次「先去文化馆」(描金幽灵)
- 右侧装置:剪纸红菱(主)+ 筊杯金月 ×2 + 英歌双槌 + 金环,漂浮视差;朱红光晕铺底
- 底部:滚动提示 `向下滚动 · 逛三件展品`

### S2 · 展品序列(〇一 三件可玩,三道门径)

每个游戏 = 全宽展签,左右交替,字段:编号(`展品 No.01 · 技术标签`)/ 名称 / 一句话卖点 / 双入口(开始玩 → / 文化导读 →)。大视觉第一版用「符号 SVG + 渐变底」,AI 生成大图为后续增强项。

1. **No.01 潮汕圣杯占卜** — 双手合十,掷筊问愿——圣杯、笑杯、阴杯,看神明如何回你。| 标签:3D 物理 · 手势可选 | 导读:掷筊怎么看
2. **No.02 合槌成阵:潮汕英歌** — 听鼓落槌,随队而行——在节奏与队形中认识「中华战舞」。| 标签:节奏判定 · 横版动作 | 导读:英歌的脸谱与角色
3. **No.03 纸上生花:中国剪纸** — 跟纸灵学徒:读帖、折剪、展开,读懂一纸吉语。| 标签:折剪展开 · 图鉴收集 | 导读:纹样里的吉祥话

### S3 · 价值主张(〇二 为何在此)

壹 **在地线索** — 以潮汕掷筊、英歌与通行剪纸吉语为线索,保留地区差异,不编「标准答案」。
贰 **可玩展品** — 3D 筊杯、节奏合槌、折剪展读——用交互建立记忆,而不是只读长文。
叁 **可核来源** — 每篇导读附快速回答、FAQ 与外链来源,区分常见说法与游戏设计。

### S4 · 文化馆入口(〇三 玩完,读懂它)

三张问题式导读卡(SEO 引流位):
- 掷筊 · 圣杯怎么看?— 一平一凸为圣杯,两平为笑杯……快速回答 + 判读口诀 + 线上与庙里的差别。| 3 篇导读 →
- 英歌 · 为什么叫「战舞」?— 脸谱、槌法、队形——从水浒角色到巡游节奏,一次讲清。| 2 篇导读 →
- 剪纸 · 纹样里的吉祥话 — 莲年有余、喜上眉梢——每个纹样都是一句可以剪出来的祝福。| 2 篇导读 →

### S5 · 信任条(〇四)

指标一排:3 可玩民俗展品 / 0 下载安装 / 本地 摄像头处理 / 免费 即开即玩;右侧竖线隔开的 SITE_DISCLAIMER 原文。

### 页脚

大馆标 `REX-GAME · 民艺馆` + 导航(展品/文化馆/关于/声明)+ 版权行。

### 门面 SEO

- `<title>`:`rex-game · 可玩的民俗文化馆`(全站 template 同步检查)
- description 保留现义、措辞去廉价化;OG title/description 与之一致
- 新 OG 分享卡:1200×630,暗场 + H1 主张 + 三符号,AI 生成
- 现有 WebSite/ItemList JSON-LD 结构保留;WebSite `name` 保持 `rex-game`,`description` 与新 meta description 同步

## 8. 组件与技术要点

### 8.1 组件清单

| 组件 | 类型 | 要点 |
|---|---|---|
| SiteHeader v2 | client(菜单交互) | 首屏上透明 → 滚动后漆底毛玻璃 `backdrop-filter`;移动端汉堡 → 全屏暗场导览菜单 |
| SiteFooter v2 | server | 大馆标 + 导航 + 声明 |
| HeroInstallation | client | 漂浮装置 + 滚动视差(rAF 或 scroll-timeline,降级 `@supports`) |
| ExhibitSection | server | 全宽展签,左右交替,双入口 |
| SectionHeader v2 | server | 〇一 + 大标题 + 金线,展览标签式 |
| Reveal | client | IntersectionObserver 进场包装器,500ms expo-out |
| Button | server | 朱砂实底(主)/ 描金幽灵(次) |
| GatewayCard / TrustStrip v2 | server | S4/S5 支撑 |

JS 增量仅限 3 个 client component(SiteHeader / HeroInstallation / Reveal)。

### 8.2 过渡期外壳并存(关键)

- v2 暗场外壳应用于:**首页 + 关于页**
- 文化页(hub/topic)**保留现有 SiteHeader/museum 浅色主题**,等 spec 2 明暗双轨统一切换——避免「暗外壳 + 浅正文」的半截工程
- 实现方式:新建 `src/styles/tokens.css` + `src/styles/gallery.css`;v2 组件为新文件,旧组件暂留;`home.css` 确认无引用后删除
- 游戏页外壳不动(各自全屏体验,spec 4 处理)

### 8.3 字体自托管

现走 Google Fonts CDN,Gitee Pages 国内访问慢且不稳。改为自托管思源宋体/黑体 woff2 **子集**(仅所需字重,放 `public/fonts/`),`@font-face` + `font-display: swap`,Hero 字重 preload。

### 8.4 图片管线

AI 生成大图 → 构建前压缩为 webp(avif 可选)→ `public/assets/`;`<img loading="lazy">`,Hero 视觉 preload。展品大视觉第一版用「符号 SVG + 渐变」,AI 大图为增强项(不阻塞上线)。

### 8.5 不变项

静态导出、Gitee Pages、`robots.ts`/`sitemap.ts`、文化页数据驱动模式(CulturePage registry)、三游戏内部代码。
